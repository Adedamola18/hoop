# Ralph Progress Log

## Codebase Patterns
- Build verify: swiftc -typecheck -sdk $(xcrun --show-sdk-path) -target arm64-apple-macos14.0 <files>
- Always update project.pbxproj when adding new Swift files (PBXFileReference + PBXBuildFile + PBXGroup + PBXSourcesBuildPhase)
- Use @Observable (Observation framework) not @Published/ObservableObject (macOS 14+)
- LSUIElement agent: Info.plist with LSUIElement=true, no dock icon
- MenuBarExtra + Settings scene pattern for menu bar app
- SMAppService as sole source of truth for launch-at-login state (no UserDefaults)
- SettingsLink is built-in SwiftUI, no need for SettingsAccess SPM package
- NSScreen notch detection: use auxiliaryTopLeftArea/auxiliaryTopRightArea
- Xcode.app is NOT installed — only CLI tools. Use swiftc -typecheck, not xcodebuild build.
- NSTrackingArea with .inVisibleRect auto-updates on view resize — no need to manually recalculate rect
- NotchPanel.notchState is set by NotchWindowManager after panel creation
- SourceKit single-file diagnostics are noise — verify with multi-file swiftc -typecheck
- NSEvent.addGlobalMonitorForEvents fires only for events outside app windows — use for click-outside dismiss
- Panel must be makeKey() to receive keyDown events (Escape key handling)

## 2026-03-10 - US-005
- Implemented custom NotchShape with concave cutout matching hardware notch profile
- Files created: Hoop/Views/NotchShape.swift
- Files changed: Hoop/Views/NotchRootView.swift, Hoop.xcodeproj/project.pbxproj
- NotchShape is a SwiftUI Shape with animatable cornerRadius, notchWidth, notchDepth
- Concave cutout uses quad Bezier curves at all notch corners for smooth transitions
- On non-notch Macs, falls back to simple RoundedRectangle
- When collapsed (notchDepth=0 or notchWidth >= rect width), draws plain rounded rect
- NotchRootView passes notchDepth=0 when collapsed, collapsedSize.height when expanded for smooth animated cutout appearance
- **Learnings for future iterations:**
  - SwiftUI Shape animatableData with nested AnimatablePair supports multi-property animation
  - NotchShape gracefully degrades: if notch dimensions exceed rect, falls back to rounded rect (handles collapsed state automatically)
  - collapsedSize in NotchState doubles as hardware notch dimensions for the cutout
